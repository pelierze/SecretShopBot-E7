# 에픽세븐 비밀상점 자동화 매크로

에픽세븐 비밀상점을 자동으로 갱신하고, 신비의 메달과 성약의 책갈피를 찾으면 자동으로 구매하는 Windows용 매크로입니다.

## 주요 기능

- 설정한 횟수만큼 비밀상점 자동 갱신
- 이미지 인식을 통한 아이템과 버튼 탐색
- 신비의 메달, 성약의 책갈피 자동 구매
- 구매 완료 검증 후 실패 시 자동 중지
- 실시간 로그와 진행 통계 표시
- ADB 기반 비활성 매크로 동작
- 원격 스크립트 동기화를 통한 GUI와 매크로 설정 자동 반영

## 실행 환경

- Windows 10 이상 권장
- ADB 연결을 지원하는 앱플레이어
- 권장 해상도 1280x720, 240dpi

Python과 ADB는 별도로 설치하지 않아도 됩니다. 배포 파일에는 실행에 필요한 구성과 ADB 도구가 포함되어 있습니다.

## 설치 방법

1. 최신 릴리스 페이지로 이동합니다.

```text
https://github.com/pelierze/SecretShopBot-E7/releases/latest
```

2. Assets에서 아래 형식의 zip 파일을 다운로드합니다.

```text
SecretShopBot-E7-v버전.zip
```

3. 원하는 폴더에 압축을 풉니다.

4. 압축을 푼 폴더 안의 실행 파일을 실행합니다.

```text
SecretShopBot-E7.exe
```

Windows SmartScreen 또는 백신 경고가 표시될 수 있습니다. 배포 출처를 확인한 뒤 실행하세요.

배포 파일의 무결성은 릴리즈 노트에 공개된 SHA256 값으로 확인할 수 있습니다.

```powershell
Get-FileHash -Algorithm SHA256 .\SecretShopBot-E7-v버전.zip
```

자세한 내용은 `SECURITY.md`를 참고하세요.

## 앱플레이어 준비

앱플레이어에서 ADB 디버깅을 켜야 합니다.

### 블루스택

1. 설정으로 이동합니다.
2. 고급 설정에서 Android Debug Bridge를 활성화합니다.
3. 기본 포트는 보통 5555입니다.

### LDPlayer

1. 설정으로 이동합니다.
2. 기타 설정에서 ADB 디버깅을 활성화합니다.
3. 기본 포트는 보통 5555입니다.

### 녹스

1. 설정으로 이동합니다.
2. Root 모드 또는 ADB 관련 설정을 활성화합니다.
3. 기본 포트는 보통 62001입니다.

## 사용 방법

1. 앱플레이어에서 에픽세븐을 실행합니다.
2. 게임에서 비밀상점 화면을 열어둡니다.
3. `SecretShopBot-E7.exe`를 실행합니다.
4. `장치 검색` 버튼을 눌러 연결 가능한 ADB 장치를 찾습니다.
5. 장치 목록에 표시된 앱플레이어를 선택합니다.
6. 선택한 장치 정보가 IP 주소와 포트 입력칸에 반영되면 `연결` 버튼을 누릅니다.
7. 리프레시 횟수와 구매 완료 검증 횟수를 확인합니다.
8. 이미지 매칭 정확도를 필요한 경우 조정합니다.
9. `시작` 버튼을 눌러 매크로를 실행합니다.

## 버튼 설명

- 장치 검색: 현재 연결 가능한 ADB 장치를 검색합니다.
- 연결: 장치 목록에서 선택된 앱플레이어에 연결합니다.
- 연결 해제: 현재 ADB 연결을 해제합니다.
- 시작: 설정값으로 매크로를 시작합니다.
- 일시정지: 실행 중인 매크로를 잠시 멈춥니다.
- 재개: 일시정지된 매크로를 다시 실행합니다.
- 중지: 매크로를 중지합니다.
- 이미지 테스트: 현재 화면에서 선택한 이미지가 인식되는지 확인합니다.

## 매크로 동작 방식

1. 현재 비밀상점 화면을 캡처합니다.
2. 첫 번째 페이지에서 목표 아이템을 찾습니다.
3. 아이템이 있으면 같은 줄의 구입 버튼을 찾습니다.
4. 구입 버튼과 구매 버튼을 순서대로 클릭합니다.
5. 비활성화된 구입 버튼을 확인해 구매 완료 여부를 검증합니다.
6. 첫 번째 페이지에 목표 아이템이 없으면 두 번째 페이지로 스크롤합니다.
7. 두 번째 페이지도 같은 방식으로 검사합니다.
8. 다음 회차가 남아 있으면 상점을 갱신합니다.

## 원격 스크립트 동기화

프로그램은 실행 시 온라인의 원격 스크립트를 확인합니다.

```text
https://raw.githubusercontent.com/pelierze/SecretShopBot-E7/master/remote_script.json
```

원격 스크립트로 다음 값을 바꿀 수 있습니다.

- GUI 제목, 섹션명, 버튼명, 라벨명
- 기본 리프레시 횟수
- 구매 완료 검증 횟수
- 이미지별 매칭 정확도
- 스와이프 위치와 시간
- 구매 대상 아이템
- 아이템과 버튼 이미지 파일명
- 구매, 갱신, 검증 대기 시간
- 구매 버튼 후보 탐색 기준
- 실행 가능한 매크로 목록
- JSON steps 기반 매크로 동작 순서

원격 스크립트는 JSON 데이터만 사용합니다. 원격 Python 코드나 명령은 실행하지 않습니다.

동기화에 실패하면 캐시된 스크립트, 내장 스크립트, 기본값 순서로 실행됩니다.

## 이미지 파일

배포 파일에는 기본 이미지가 포함되어 있습니다.

기본 이미지 위치는 아래와 같습니다.

```text
images/items
images/buttons
```

이미지 인식이 잘 되지 않으면 동일한 파일명으로 이미지를 다시 캡처해 교체할 수 있습니다.

기본 파일명:

- `images/items/mystic_medal.png`
- `images/items/covenant_bookmark.PNG`
- `images/buttons/refresh_button.png`
- `images/buttons/confirm_button.png`
- `images/buttons/purchase_button.png`
- `images/buttons/buy_button.png`
- `images/buttons/purchase_button_disabled.png`

## 원격 매크로 방식

배포된 exe는 ADB 연결, 화면 캡처, 이미지 매칭, 클릭, 스와이프를 수행하는 구동기 역할을 합니다.

매크로 목록과 일부 동작 정의는 `remote_script.json`에서 불러옵니다. 현재 비밀상점 매크로는 내장 실행기로 동작하며, 이후 추가 매크로는 JSON steps 방식으로 정의할 수 있습니다.

지원되는 JSON steps 액션:

- `log`
- `wait`
- `screenshot`
- `tap_image`
- `swipe`
- `repeat`

보안을 위해 원격 Python 코드, 셸 명령, 임의 파일 경로는 실행하지 않습니다.

## 문제 해결

### 연결이 안될 때

- 앱플레이어가 실행 중인지 확인합니다.
- 앱플레이어의 ADB 디버깅이 켜져 있는지 확인합니다.
- 장치 검색 결과에 앱플레이어가 표시되는지 확인합니다.
- 목록에서 올바른 장치를 선택했는지 확인합니다.
- 방화벽 또는 백신이 연결을 차단하는지 확인합니다.

### 이미지 인식이 안될 때

- 앱플레이어 해상도가 1280x720인지 확인합니다.
- 이미지 테스트 기능으로 현재 화면과 이미지의 매칭률을 확인합니다.
- 이미지 매칭 정확도를 낮춰 테스트합니다.
- 필요한 경우 이미지 파일을 현재 화면 기준으로 다시 캡처합니다.

### 구매 후 중지될 때

- 골드가 부족한지 확인합니다.
- 구매 완료 검증 이미지가 현재 게임 화면과 맞는지 확인합니다.
- `purchase_button_disabled.png` 이미지를 다시 캡처해 교체합니다.
