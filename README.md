# 에픽세븐 비밀상점 자동화 매크로

에픽세븐 게임의 비밀상점을 자동으로 리프레시하고 원하는 아이템을 구매하는 매크로입니다.

## 주요 특징

- 🔄 **자동 상점 리프레시**: 설정한 횟수만큼 비밀상점 자동 리프레시
- 🔍 **이미지 인식**: OpenCV를 사용한 정확한 아이템 인식
- 🛍 **자동 구매**: 신비의 메달, 성약의 책갈피 자동 구매
- 📊 **실시간 통계**: 구매 내역 및 진행 상황 실시간 표시
- 🎮 **사용자 친화적 GUI**: 직관적인 인터페이스
- ✨ **비활성 매크로**: ADB를 사용하여 매크로 동작 중에도 키보드/마우스 자유롭게 사용 가능

## 필수 요구사항

### 1. Python 설치
- Python 3.8 이상
- [Python 공식 홈페이지](https://www.python.org/downloads/)에서 다운로드

### 2. ADB (Android Debug Bridge)
- Android SDK Platform Tools 필요
- [다운로드 링크](https://developer.android.com/studio/releases/platform-tools)
- 다운로드 후 압축 해제하고 `adb.exe` 경로를 시스템 환경 변수 PATH에 추가

**🔑 ADB를 사용하는 이유:**
- **비활성 매크로**: 매크로가 동작하는 동안 사용자가 키보드와 마우스를 자유롭게 사용할 수 있습니다.
- 앱플레이어와 독립적으로 동작하여 다른 작업 방해 없음
- 안정적인 화면 캡처 및 입력 제어

### 3. 앱플레이어 (에뮬레이터)
- 블루스택, LDPlayer, 녹스 등 ADB 연결을 지원하는 앱플레이어
- 앱플레이어 설정에서 ADB 디버깅 활성화 필요

## 이미지 파일 준비

매크로가 작동하려면 다음 이미지 파일들이 필요합니다:

### 아이템 이미지 (`images/items/` 폴더에 저장)
- `mystic_medal.png`: 신비의 메달 아이템 이미지
- `covenant_bookmark.png`: 성약의 책갈피 아이템 이미지

### 버튼 이미지 (`images/buttons/` 폴더에 저장)

**상점 갱신 프로세스 (갱신 → 확인 2단계):**
- `refresh_button.png`: 갱신 버튼 (상점 리프레시 시작)
- `confirm_button.png`: 확인 버튼 (갱신 확인)

**아이템 구매 프로세스 (구입 → 구매 2단계):**
- `purchase_button.png`: 구입 버튼 (1단계)
- `buy_button.png`: 구매 버튼 (2단계, 최종 구매)

### 이미지 캡처 방법
1. 게임을 실행하고 비밀상점 화면으로 이동
2. 각 요소(아이템, 버튼)를 스크린샷으로 캡처
3. 이미지 편집 프로그램으로 필요한 부분만 정확히 잘라내기
4. PNG 형식으로 저장하고 위의 경로에 배치

**💡 팁**: 이미지는 가능한 한 작고 명확하게 캡처하세요. 배경이 포함되지 않도록 주의하세요.

## 사용 방법

### 1. 앱플레이어 실행 및 ADB 연결 확인
```bash
# ADB가 제대로 설치되었는지 확인
adb version

# 연결된 디바이스 확인
adb devices
```

### 2. 프로그램 실행
```bash
python main.py
```

### 3. GUI 사용 방법

1. **ADB 연결**
   - IP 주소: 기본값 `127.0.0.1` (로컬)
   - 포트: 앱플레이어별 기본 포트
     - 블루스택: `5555`
     - LDPlayer: `5555` 또는 `5037`
     - 녹스: `62001`
   - "연결" 버튼 클릭

2. **매크로 설정**
   - 리프레시 횟수: 상점을 리프레시할 총 횟수 입력
   - 아이템당 구매 횟수: 원하는 아이템을 발견했을 때 구매할 개수 입력

3. **실행**
   - 게임에서 비밀상점 화면을 열어둔 상태에서 "▶ 시작" 버튼 클릭
   - 실시간으로 로그와 통계 확인
   - 중지하려면 "■ 중지" 버튼 클릭

## 앱플레이어별 ADB 연결 방법

### 블루스택 (BlueStacks)
1. 블루스택 설정 → 고급 → Android Debug Bridge 활성화
2. 포트: `5555`
```bash
adb connect 127.0.0.1:5555
```

### LDPlayer
1. LDPlayer 설정 → 기타 설정 → ADB 디버깅 활성화
2. 포트: `5555`
```bash
adb connect 127.0.0.1:5555
```

### 녹스 (Nox)
1. 녹스 설정 → Root 모드 활성화
2. 포트: `62001`
```bash
adb connect 127.0.0.1:62001
```

## 동작 원리

1. **상점 스캔**: ADB를 통해 현재 상점 화면을 스크린샷으로 캡처
2. **이미지 매칭**: OpenCV를 사용하여 원하는 아이템 탐색
3. **페이지 전환**: 첫 페이지에서 아이템이 없으면 드래그하여 두 번째 페이지로 이동
4. **구매 프로세스 (구입 → 구매 2단계)**: 
   - 아이템 발견 시 클릭
   - **1단계**: 구입 버튼 클릭
   - **2단계**: 구매 버튼 클릭 (최종 구매)
   - 설정한 횟수만큼 반복
5. **갱신 프로세스 (갱신 → 확인 2단계)**: 
   - **1단계**: 갱신 버튼 클릭
   - **2단계**: 확인 버튼 클릭
   - 설정한 횟수만큼 전체 과정 반복

### 비활성 매크로의 장점
- ADB를 사용하여 앱플레이어에 직접 명령을 전송
- 매크로 실행 중에도 PC에서 다른 작업 가능
- 키보드와 마우스를 자유롭게 사용 가능

## 문제 해결

### ADB 연결이 안될 때
- 앱플레이어가 실행 중인지 확인
- 방화벽이 연결을 차단하는지 확인
- 앱플레이어를 재시작 후 다시 시도

### 이미지 인식이 안될 때
- 이미지 파일이 올바른 경로에 있는지 확인
- 이미지가 너무 크거나 작지 않은지 확인
- 게임 해상도와 이미지 해상도가 일치하는지 확인

### 구매가 제대로 안될 때
- 로그를 확인하여 어느 단계에서 실패하는지 파악
- 버튼 이미지를 다시 캡처
- 대기 시간(`delay`) 값을 늘려보기
